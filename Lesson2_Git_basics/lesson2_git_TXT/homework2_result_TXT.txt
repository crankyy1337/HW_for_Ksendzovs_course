TXT
 1. Создать внешний репозиторий c названием TXT.
	зайти в свой профиль на гитхабе -> перейти в repositories -> кликнуть new -> создать публичный репозиторий "lesson2_git_TXT"  (по желанию можно добавить README файл) //мне такое название удобнее

 2. Клонировать репозиторий TXT на локальный компьютер.
	git clone https://github.com/crankyy1337/lesson2_git_TXT.git

 3. Внутри локального TXT создать файл “new.txt”.
	cd lesson2_git_XML && touch new.txt

 4. Добавить файл под гит.
	git add new.txt

 5. Закоммитить файл.
	git commit -m "added new.txt for homework"

 6. Отправить файл на внешний GitHub репозиторий.
	git push
 7. Отредактировать содержание файла “new.txt” - написать информацию о себе (ФИО, возраст, количество домашних животных, будущая желаемая зарплата). Всё написать в формате TXT.
	nano new.txt	//открываем на редактирование
ФИО: Мурадели Георгий
Возраст: 26
Кол-во домашних животных: 1
Будущая зарплата: 4000$
	CTRL+X -> y -> ENTER	//выходим с сохранением изменений

 8. Отправить изменения на внешний репозиторий.
	git add new.txt
	git commit -m "added bio"
	git push

 9. Создать файл preferences.txt
 10. В файл preferences.txt” добавить информацию о своих предпочтениях (Любимый фильм, любимый сериал, любимая еда, любимое время года, сторона которую хотели бы посетить) в формате TXT.
	cat >> preferences.xml	//создаем и входим в режим редактирования
Любимый фильм: marvel
Любимый сериал: Менталист
Любимая еда: wok
Любимая страна: Япония
CTRL+D	//выходим с сохранением изменений

 11. Создать файл sklls.txt добавить информацию о скиллах которые будут изучены на курсе в формате TXT
	cat >> skills.txt
Скиллы: Терминал, гитбаш, постман, основы javascript, и т д
CTRL+D	//выходим с сохранением изменений

 12. Сделать коммит в одну строку.
 13. Отправить сразу 2 файла на внешний репозиторий.
	git add skills.txt preferences.txt && git commit -m "Добавлены скиллы и предпочтения"
	git push

 14. На веб интерфейсе создать файл bug_report.txt.
 15. Сделать Commit changes (сохранить) изменения на веб интерфейсе.
	перейти в нужный репозиторий на github -> нажать add file -> в выпадающем меню выбрать create new file -> ввести его имя (bug_report.txt) и содержание -> внизу страницы нажать commit new file (остальные опции можно не изменять)

 16. На веб интерфейсе модифицировать файл bug_report.txt, добавить баг репорт в формате TXT.
 17. Сделать Commit changes (сохранить) изменения на веб интерфейсе.
	перейти в нужный репозиторий на github -> перейти в файл bug_report.txt -> в секции с содержимым файла нажать на пиктограмму "карандаш" (edit this file) -> внести изменения -> внизу страницы нажать commit new file (остальные опции можно не изменять)

 18. Синхронизировать внешний и локальный репозиторий TXT
	git pull
